Desarrollo del Prototipo

Origen del Prototipo

El prototipo inicial fue construido utilizando como base la plataforma Lovables. A partir de esta base se realizaron diversas modificaciones y personalizaciones manuales hasta alcanzar la versión actual que se presenta como demostración funcional del proyecto.

El objetivo de este prototipo es mostrar de forma clara la dirección conceptual del sistema y permitir visualizar cómo sería la experiencia inicial del usuario dentro de la plataforma.

Desarrollo de la Aplicación

En una primera etapa se exploró el uso de AntiGravity de Google como base para el desarrollo de la aplicación. Sin embargo, el consumo gratuito de modelos de lenguaje (LLM) dentro de esta plataforma resultó demasiado costoso para una fase experimental, lo que limitó significativamente su utilización.

Debido a esta situación, se optó por desarrollar gran parte del sistema desde cero, utilizando TypeScript junto con React 19 como base para la estructura de la aplicación.

Reorganización de Arquitectura

Durante el proceso de desarrollo fue necesario realizar una reorganización completa de la arquitectura del proyecto. Tanto las herramientas utilizadas inicialmente —como LocalBuild y Antigravity— generaban estructuras de archivos y carpetas poco eficientes para un desarrollo escalable.

Por este motivo, se procedió a reestructurar manualmente la organización del proyecto con el objetivo de mejorar la claridad del código, facilitar el mantenimiento y preparar la base del sistema para una arquitectura más robusta en futuras etapas de desarrollo.

Implementación del Prototipo

El prototipo presentado actualmente es una simplificación deliberada de la experiencia final del usuario. Su objetivo no es representar el sistema completo, sino demostrar la lógica principal y el flujo conceptual de la idea.

Técnicamente, el prototipo funciona como una página web simple ejecutada con JavaScript. Sin embargo, durante el proceso de desarrollo fue escrito utilizando TypeScript para mantener una estructura más cercana a la arquitectura final que se pretende implementar.

Esta versión extremadamente básica permite simular algunas interacciones clave del sistema sin requerir todavía la infraestructura completa que será necesaria en etapas posteriores.

Diseño Responsivo

Durante el desarrollo del prototipo también se implementó un enfoque de diseño responsivo. Esto permitió validar el funcionamiento y la experiencia de usuario tanto en dispositivos móviles como en entornos de escritorio (desktop).

La adaptación del diseño a distintos tamaños de pantalla facilita evaluar desde etapas tempranas cómo se comportará la interfaz del sistema en diferentes dispositivos, algo especialmente relevante para aplicaciones vinculadas al comercio electrónico.

Producción de Material Audiovisual

Para la generación y edición del material audiovisual utilizado en la presentación del prototipo se emplearon herramientas de inteligencia artificial en combinación con Adobe Premiere.

Este proceso permitió acelerar la producción de contenido visual para explicar el funcionamiento de la idea y facilitar su comunicación durante las etapas iniciales del proyecto.
