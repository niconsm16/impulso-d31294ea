Elementos que ejecutará

Elemento Función Consumo promedio
Servidor Linux ligero (Alpine / Arch minimalista) Sistema operativo base del servidor RAM: 80-150MB | CPU: <1% idle
Docker Swarm / K3s Orquestación de contenedores y servicios RAM: 200-400MB | CPU: 1-3%
Bases de datos locales Persistencia y acceso a información del sistema RAM combinada aproximada: 1-2GB dependiendo del uso
Agentes Zero Claw Agentes ligeros para automatización y delegación ~5MB RAM por instancia
LLM 8B-20B Modelo de lenguaje para razonamiento y herramientas Dependiente de GPU/VRAM

Base de Datos Complementaria

Base de datos Función Consumo promedio
DynamoDB Local DB documental para catálogo RAM: 300-800MB | CPU: bajo | Storage dependiente del catálogo
Redis Caché de consultas frecuentes RAM: 200MB-1GB según dataset | CPU muy bajo
Postgres + pgvector Búsqueda semántica mediante embeddings RAM: 500MB-1.5GB | CPU moderado en consultas vectoriales
ScyllaDB (escala futura) Reemplazo de Redis para alta concurrencia Limitado mediante flag --memory del proceso para evitar consumo total del host.

Procesamiento Asincrónico

RabbitMQ gestiona las tareas asincrónicas del sistema.

  • Pedidos de clientes entran en cola
  • La IA completa información progresivamente
  • Normalización automática de productos
  • Procesamiento fuera del flujo principal
